Goal
Create a production-style project that does all of the following:
Detect one or two hands in real time from webcam video.
Extract hand landmarks and interpret the current hand sign.
Support custom user-defined gestures by recording samples and training on them.
Recognize multi-step gesture combinations such as sequences and two-hand combinations.
Generate a visual animation that shows how the detected gesture or combination was formed step by step.
Be modular, clean, and implemented in phases so each stage works before moving to the next.
Technical direction
Use MediaPipe Hand Landmarker / Gesture Recognizer for real-time hand landmark detection because it supports real-time gesture recognition and exposes hand landmarks for web and app pipelines.
Use OpenCV for webcam capture, frame processing, overlays, and recording datasets.
For custom gesture classification, start with a simple landmark-based classifier first, such as RandomForestClassifier from scikit-learn, then keep the code structured so it can later be upgraded to a PyTorch deep learning model. scikit-learn documents RandomForestClassifier and good practice around reproducibility with random_state.
For gesture sequences and combinations, add a second-stage temporal model using PyTorch, preferably an LSTM or sequence classifier on top of landmark vectors over time. PyTorch is the recommended deep learning framework here and its docs show the standard model-building workflow with torch.nn.Module.
For 3D or polished visual playback of gesture formation, use Three.js with glTF-ready animation support for the frontend visualization module.

Create a clear schema for gesture samples. Each sample should include:
sample_id
gesture_label
frame_sequence or landmark_sequence
handedness
num_hands
normalized_landmarks
raw_landmarks
timestamp
source_video or webcam_session_id
Model design
Implement two model layers:
Static gesture model
Input: flattened normalized landmarks for one frame
Output: single gesture class
Baseline: RandomForestClassifier
Keep interface generic so another model can replace it later
Temporal / combination model
Input: sequence of landmark frames over time
Output: gesture sequence or combination class
Baseline: rule engine
Upgrade: PyTorch LSTM or GRU classifier
Important implementation details
Normalize landmarks relative to wrist and hand scale
Handle missing hand frames gracefully
Separate left and right hand pipelines
Add debounce logic for stable gesture output
Use confidence thresholds
Add calibration mode for user-specific gestures
Make all thresholds configurable from a settings file
